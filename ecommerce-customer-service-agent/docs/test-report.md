# 测试报告

## 当前自动化用例

| 用例 | 预期结果 |
|---|---|
| 退货规则问题 | 进入RAG路径 |
| 投诉风险表达 | 转人工处理 |
| 查询发货但缺订单号 | 追问订单号 |
| 下一轮只提供订单号 | 恢复上一轮Tool并执行 |
| 不同session提供订单号 | 不继承他人pending action |
| 他人查询DD1001 | 返回ACCESS_DENIED |
| 物流查询 | 调用物流Tool |
| 库存查询 | 调用库存Tool |
| 模型返回删除订单工具 | 被白名单阻止 |

## 验收标准

- 测试必须由程序断言结果，不能只观察终端是否报错。
- Tool结果必须包含 `success/error_code/message/data`。
- 失败路径不得编造订单、物流、库存或退款信息。
- 会话与用户权限测试必须独立存在。
